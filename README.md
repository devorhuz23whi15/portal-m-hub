# portal-m-hub

## 项目简介

portal-m-hub 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的集合与分发点，方便维护和引用多个互不关联的前端页面。

## 目录说明

```
portal-m-hub/
├── pages/          # 存放各个独立 HTML 页面（按子目录或文件组织）
├── assets/         # 公共资源文件（如 CSS、JS、图片等，可选）
├── index.html      # 首页或导航页面（可选）
└── README.md       # 本文件
```

- `pages/`：每个独立页面可放置于单独的子文件夹中，或直接以文件形式存放，便于管理和索引。
- `assets/`：如果多个页面共享资源（如样式、脚本、字体），可统一放置于此目录。
- `index.html`：可选，用于提供页面列表或简单导航。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，彼此之间无依赖关系。每个页面可视为一个独立的“微站点”，可通过路径直接访问。归档目的包括但不限于：

- 保存静态页面快照
- 发布临时或实验性页面
- 提供可复用的前端片段

如需添加新页面，请遵循以下惯例：

- 在 `pages/` 下创建以页面名称命名的文件夹（或直接放置文件）。
- 确保页面内引用的资源路径正确，推荐使用相对路径。
- 如有必要，可在页面文件夹内附带说明文件（如 `README.md`）。

## 维护说明

- 本仓库主要用于归档和发布，不建议频繁修改已归档页面。
- 如需更新，请提交 Pull Request 或直接推送至主分支（根据仓库权限而定）。
- 请保持页面内容简洁，避免包含无关外部资源或跟踪代码。
- 若页面不再需要，可将其从 `pages/` 移除，并更新相关索引。

## 许可

本仓库中的内容遵循默认的开源许可（如未特别说明，可视为 MIT 协议）。每个独立页面可附带自己的许可声明，以页面内标注为准。
